# حالة مشروع منصة آفاق الرقمية

التاريخ: 11 يوليو 2026 — الإصدار 1.0.0

## يعمل فعليًا دون خدمة خارجية

- الهوية العربية RTL المتجاوبة، الوضع الداكن والفاتح، الهيدر والتذييل وتوقيع عدناني ورابط X الموحد.
- الصفحة الرئيسية، مكتبة الأدوات، التصنيفات، من نحن، التواصل، الخصوصية، الشروط، الباقات، الإعداد، Health Check وصفحات الأخطاء.
- محرر الصور: فتح صورة، نص عربي وخطوط، إضافة صورة، سحب، تحجيم، تدوير، ترتيب الطبقات، قص تفاعلي، PNG/JPG.
- معالجة الصور الدُفعية: حتى 25 صورة، تغيير مقاس، جودة، PNG/JPG/WEBP وZIP.
- العلامة المائية الجماعية: نص أو شعار، حجم، شفافية، 9 مواضع وZIP.
- QR: رابط، نص، هاتف، بريد، Wi‑Fi، ألوان، شعار في PNG وتصدير SVG.
- مختبر المطور: JSON وBase64 Unicode وUUID وSHA وURL وTimestamp وJWT Decoder بتحذير وRegex بقيود.
- وضع استعراضي للوحة العضو والمالك دون قاعدة بيانات.
- عداد تجريبي من الخادم مع تصريح موقع، Cookie موقع وخصم بعد النجاح فقط.

## يعمل بعد ربط Supabase

- التسجيل وتأكيد البريد والدخول والخروج واستعادة كلمة المرور.
- ملفات الأعضاء والأدوار والباقات والرصيد وسجل الاستخدام.
- RLS ودوال الخصم الذرية وIdempotency.
- لوحة OWNER الفعلية، البحث، الأدوار، الحظر، Credits، تصفير حد الاستخدام، CSV وAudit Log.
- حماية OWNER في الخادم وTrigger داخل PostgreSQL.
- MFA/TOTP لحساب OWNER.
- نموذج التواصل وحفظ رسائل الدعم.
- رابط X القابل للتعديل من لوحة OWNER.

## يحتاج مفتاحًا خارجيًا

- إزالة الخلفية: `REMOVE_BG_API_KEY`. الـ Adapter ومسار الخادم وفحص الملف والمعاينة والتنزيل منفذة.
- Cloudflare Turnstile: يعمل عند وضع Site Key وSecret، ويتخطى التحقق فقط إذا لم يهيأ في التطوير.
- SMTP: يدار عبر Supabase Auth أو SMTP خاص بالنطاق.

## مجهز معماريًا ومخفي

- المتجر: جداول المنتجات والطلبات والعناصر والمدفوعات والفواتير والكوبونات، و`MARKETPLACE_ENABLED=false`.
- Developer API: ملفات المطورين ومفاتيح Hash والاستخدام والحدود، و`DEVELOPER_API_ENABLED=false`.
- باقات مدفوعة: الجداول والحدود والاشتراكات موجودة، لكن لا أسعار ولا بوابة دفع.

## مؤجل بوضوح

- PDF Studio وOCR عربي وماسح المستندات وفحص سلامة الصور وأدوات الطلاب والشركات.
- بوابة دفع وفوترة حقيقية.
- Rate Limiting موزع بين عدة خوادم؛ الإصدار الحالي يستخدم ذاكرة العملية للطلبات وPostgreSQL للعداد الدائم.
- تخزين ملفات دائم؛ غير مطلوب للأدوات الحالية ولا تُحفظ صور المستخدم افتراضيًا.
- اختبارات متصفح End-to-End مع Supabase حقيقي ومفاتيح خارجية؛ تحتاج بيئة Staging منفصلة.
