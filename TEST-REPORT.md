## Smoke Test للتشغيل

- `/` أعاد HTTP 200.
- `/tools` أعاد HTTP 200.
- `/admin` في الوضع الاستعراضي أعاد HTTP 200.
- `/api/health` أعاد HTTP 200 وحالة `ok`.
- `/setup` أعاد HTTP 200.
- تشغيل الحزمة المستقلة بواسطة `npm start` نجح، والصفحة الرئيسية أعادت HTTP 200.
- في وضع الإنتاج دون `DEMO_MODE=true` أعادت `/admin` تحويل HTTP 307 إلى تسجيل الدخول، ولم تعرض بيانات العرض.
- تكامل الحد: العمليات 1 و2 و3 نجحت، والمتبقي أصبح 2 ثم 1 ثم 0، والتصريح الرابع أعاد HTTP 402 مع `limit_reached`.

الاختبارات تغطي:

- السماح بثلاث عمليات ناجحة فقط ومنع الرابعة.
- عدم خصم العملية المكررة بفضل Idempotency.
- ثبات العداد عند فشل الأداة وعدم إرسال Success Commit.
- عدم قدرة OWNER أو ADMIN على تعديل دور OWNER عبر الإجراءات العادية.
- حدود صلاحيات ADMIN.
- عدم تكرار Slugs ووجود الأدوات الست المطلوبة.
- حساب أبعاد الصور مع المحافظة على النسبة وبدونها.

## يحتاج بيئة Staging خارجية

- إرسال البريد وتأكيده وإعادة تعيين كلمة المرور عبر SMTP فعلي.
- سياسات RLS بحسابين على مشروع Supabase حقيقي.
- MFA وتدفق AAL2 الكامل.
- مزود remove.bg برصيد فعلي.
- Turnstile بمفاتيح نطاق Staging.
