# سياسات RLS

جميع سياسات الإصدار الأول موجودة داخل `database/migrations/001_initial_schema.sql` كي تُطبق في معاملة واحدة.

- العضو يقرأ ملفه واستخدامه ورصيده فقط.
- SUPPORT يقرأ ما يلزم للدعم دون صلاحيات مالية أو أدوار.
- ADMIN يدير الأعضاء المسموحين له، لكن Trigger قاعدة البيانات يمنعه من حذف دور OWNER أو تغييره.
- الجداول الحساسة مثل `anonymous_usage` و`admin_audit_logs` لا تملك سياسة كتابة مباشرة للمتصفح.
- دوال خصم الاستخدام وCredits مسموحة لـ `service_role` فقط، والمفتاح يبقى في الخادم.

اختبر السياسات بحسابين مختلفين قبل الإنتاج، ولا تعطّل RLS لتجاوز خطأ في الواجهة.
